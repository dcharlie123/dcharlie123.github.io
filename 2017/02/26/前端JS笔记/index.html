  • 通过（.）访问

③继承

  • 假设要查询对象o的属性x，如果o中不存在x，那么将会继续在o的原型对象中查询属性x，沿着“链”一直查找到原型是null为止。
  • 假设给对象o的属性x赋值，如果o中已经有属性x（这个属性不是继承来的）那么改变已有属性x的值;如果o不存在属性x，那么赋值操作给o添加一个新属性x;如果之前o继承自
    属性x，那么这个继承的属性将被新建的同名属性覆盖。如果o继承自一个只读属性x，那么赋值操作是不允许的。

④检测属性

  • ①in：对象的自有属性或继承属性中包含这个属性，返回true；（in能检测显式的undefined）
  • ②hasOwnProperty()：检测给定的名字是否是对象的自有属性。继承的属性将返回false
  • ③propertyIsEnumerable：检测自有属性的可以枚举属性。
  • ④Object.keys()：检测可枚举的自有属性，返回一个数组
  • ⑤Object.getOwnPropertyNames()：返回所有自有属性

属性的特性：可写性，可枚举性，可配置性

获取一个对象的属性描述 Object.getOwnPropertyDescriptor()

对于一个继承和不存在的属性，返回undefined

对这几个属性特性的配置Object.defineProperty()(不能修改继承属性)

对象三个属性

  • ①原型属性
    ES5中用Object.getPrototypeOf()可以查询对象的原型；可以用p.isPrototypeOf(o)来检验p是否是o的原型

  • ②类属性
    通过Object.prototype.toString.call(o).slice(8,-1);

  • ③可扩展性

    • Object.isExtensible()用于判断对象是否可扩展。
    • Object.preventExtensions()将对象转化为不可扩展，一旦转换就无法转回可扩展。且只影响对象本身的可扩展性。如果给不可扩展的对象的原型添加属性，这个对象仍会继承这个新属性。
    • Object.seal()除了将对象设置成不可扩展的，还可以将对象所有的自有属性设置成不可配置的（即：不能添加新属性，已有的属性不能删除或配置，但已有的可写属性依然可以配置）。可以用Object.isSealed()检测
    • Object.freeze()对象冻结：不可扩展，属性不可配置，数据只读，但存取器属性不受影响。Object.isFrozen()检测

序列化对象和反序列化

数组方法

  • slice()

    Array.slice（）方法返回指定数组的一个片段或子数组。它的两个参数分别指定了片段的开始和结束的位置。返回的数组包含第一个参数指定的位置和所有到但不含第二个参数指定的位置之间的所有数组元素。

  • splice()

    Array.splice（）方法是在数组中插入或删除元素的通用方法。不同于slice（）和concat（），splice（）会修改调用的数组。

第一个参数表示起始位置（不包括），第二个表示删除元素个数

使用instanceof的问题是在Web浏览器中有可能有多个窗口或窗体（frame）存在。每个窗口都有自己的JavaScript环境，有自己的全局对象。并且，每个全局对象有自己的一组构造函数。因此一个窗体中的对象将不可能是另外窗体中的构造函数的实例。窗体之间的混淆不常发生，但这个问题足已证明instanceof操作符不能视为一个可靠的数组检测方法。
